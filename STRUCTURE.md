## Як влаштована теорія (prep.html)

`prep.html` — згенерований файл. Вміст усіх тем живе у `generator/prep_data_*.js`.
Кожна тема — обʼєкт із такими полями:

```js
{
  p: "P0",                    // пріоритет: P0 (критично) / P1 / P2
  name: "Назва теми",         // видима назва; за нею шукаються гілки в Discussions
  what: ["абзац", "абзац"],   // блок «Що це і як працює»
  problem: "один абзац",      // блок «Яку проблему вирішує»
  where: ["пункт", ...],      // блок «Де використовується на практиці»
  code: ["рядок коду", ...],  // блок «Приклад» (масив рядків; можна [] — блок зникне)
  traps: ["пункт", ...],      // блок «Пастки та типові помилки»
  qs: ["питання", ...],       // блок «Контрольні питання»
}
```

### Додати нову тему

1. Відкрий потрібний `generator/prep_data_*.js`
2. Додай обʼєкт теми **В КІНЕЦЬ масиву `topics`** (перед закривними `],\n};`)
3. Онови число в `title` (наприклад `"SWIFT — 16 тем"`)
4. Перегенеруй і закоміть (див. «Робочий цикл» нижче)

⚠️ **Чому тільки в кінець:** id тем — порядкові (`swift-0`, `swift-1`…). Вставка
в середину зсуне id усіх наступних тем — поламаються якірні посилання з нотаток
у Discussions, збережений прогрес галочок і посилання з розкладу.

### Змінити наявну тему

Просто редагуй її поля у `prep_data_*.js` і перегенеруй. Назву (`name`) краще
не міняти без потреби — за нею привʼязані гілки нотаток у Discussions.

## Робочий цикл оновлення

```bash
cd ~/Desktop/IT/Other/ios-interview-prep
node generator/gen_prep_html.js     # перегенерує prep.html
```

Потім у GitHub Desktop: Commit to main → Push origin. Через 1–2 хв сайт оновиться.

Найпростіший шлях — попросити Claude: «додай тему про X» — він редагує дані,
генерує і лишає готові зміни для коміту.

## Інші сторінки

- **index.html** — статичні картки. Додаєш сторінку сайту → додай картку тут.
- **trainer.html** — самодостатній файл із завданнями (розмітка + стилі + JS).
  Завдання додаються редагуванням HTML за зразком сусідніх карток.
- **schedule.html** — 28 карток-днів по 4 тижнях. День = `<div class="day" data-d="wXdY">`;
  посилання на теми ведуть на `prep.html#<id-теми>`.

## Як працюють інтерактивні штуки

- **Прогрес (галочки)** — localStorage браузера, ключі: `ios-prep-deep-v1` (теми),
  `ios-prep-schedule-v1` (дні розкладу), тренажер має свій. Прогрес живе на домені
  сайту в конкретному браузері; при зміні домену — обнуляється.
- **Коментарі до тем** — giscus: кнопка «💬 Показати коментарі» вантажить віджет,
  привʼязка гілки — за назвою теми (data-term). Гілки зберігаються у
  GitHub Discussions репозиторію (категорія Announcements для віджета,
  General — для нотаток через 💬/🧵 у заголовку теми).
- **Картинки в нотатках** — через GitHub: відкрий гілку (🧵) і встав скріншот
  (Cmd+V) у коментар на github.com; на сайті він відобразиться.
- **Редагувати/видалити коментар** — клік на дату коментаря у віджеті →
  відкриється на GitHub → меню ··· → Edit / Delete.

## Деплой і якщо він упав

Сайт публікує GitHub Actions (workflow `static.yml`, Settings → Pages →
Source: GitHub Actions). Статус: https://github.com/Nazarzbs/iOSprep/actions

**Якщо деплой упав** («Deployment failed», «Multiple artifacts»): НЕ натискай
Re-run — зроби будь-який новий коміт (пробіл у README) і Push. Перезапуск
через Re-run створює дублікат артефакту і падає знову.
